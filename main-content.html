This very codeblock resides in a file called main-content.html. It's included like so:

        
<script>
  render('main-content.html', document.currentScript);
</script>

The render function takes two parameters, the url to the module, a html file, and the reference to the current script block. The latter is required for the module loader to correctly replace the script block with the loaded module.

The file main-content.html contains two tags: a html element tag and a script tag. They must be written in that exact order. The script tag then needs to call the automagically accessible done() function, to tell the module loader that it's finished initializing.

            
    <div>
      Content goes here.
    </div>
    
    <script>
      // element, context and done are accessible through the module loader.
      element.innertHTML = `Element's a reference to the codeblock above!`;
      context.ihazmodules = true;
      done();
    </script>

When all modules are loaded (has called their done functions) the module loader will call a function called bootstrap, if defined, and pass in a variable called context. The context variable is also accessible from within the modules. Either use that to pass data, or do it globally. I don't give a fuck.
